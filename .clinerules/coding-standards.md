# Coding Standards

Follow existing project conventions before introducing new patterns.

## General
- Prefer small, incremental changes
- Keep functions focused
- Avoid dead code and commented-out code
- Do not add dependencies unless necessary

## Structure
- Reuse existing utilities/helpers when possible
- Keep related logic near existing feature files
- Match existing naming conventions and file organization

## Error handling
- Fail clearly
- Do not swallow errors silently
- Return actionable error messages where appropriate

## Documentation
- Add inline comments only where the intent is not obvious
- Update documentation when behavior changes
- Keep examples accurate and runnable when possible
